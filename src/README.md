# Шаг 1: вынос данных и БД-слоя из монолита

_Пересобрано на актуальной версии файла (`FitnessRPG_42_balance_rebalance_4.jsx`, 15 559 строк).
Зона данных (первые 3372 строки — весь баланс, БД-слой, стили) оказалась **byte-for-byte
идентична** предыдущей версии. Значит все правки с прошлого раза были только внутри
`AppInner`/подкомпонентов — они целиком попали в новый `App.jsx`._

Что сделано: из одного файла вынесены все "чистые" данные, правила баланса
и функции работы с Supabase — **без единой смысловой правки**. Логика
(`AppInner`, подкомпоненты, JSX) не тронута вообще, только вырезана и обёрнута импортами.

## Проверка перед выдачей
- Собрано esbuild-бандлом (`--bundle`, external react/lucide-react) — **0 ошибок, 0 предупреждений**.
- Построчная сверка мультимножеством строк с оригиналом:
  - `App.jsx` + `styles.js` — совпадают **1-в-1** (12183 = 12183 строки).
  - Вынесенные данные (`lib/`, `game/`) — ничего лишнего не появилось (0 задублированных строк)
    и ничего реально не потеряно (все "расхождения" — вводные комментарии, которые
    физически на месте, просто грубая проверка приняла их за "шапку" файла).
- Единственное сознательное перемещение: `MOCK_GUILD_MEMBERS` (пустой массив-заглушка)
  переехал из "AVATAR ICONS" в `game/guild.js`, куда он смыслово относится.

## Как залить в твой репозиторий (GitHub через сайт)
1. Зайти в репозиторий → **Add file → Upload files**.
2. Перетащить туда всю папку `FitnessRPG_step1_data_extraction` целиком —
   современный GitHub сохраняет вложенность `lib/` и `game/` при перетаскивании папки.
3. Убедиться, что `App.jsx` встал на место старого файла (замени/удали старый монолитный
   файл, если он лежал под другим именем — иначе останутся оба и надо явно указать,
   какой из них служит точкой входа).
4. Закоммитить прямо в `main` — Vercel подхватит пуш и соберёт как обычно.

**Рекомендация (не обязательно для этого шага, но по силе безопасности):**
раз это уйдёт сразу тестировщикам, лучше сначала залить в отдельную ветку и
открыть Pull Request — Vercel сам соберёт отдельный preview-URL, на котором можно
всё проверить до того, как это увидят тестировщики. Могу расписать это отдельно,
если пригодится.
